## Deviations from Original Story Scope

Logged for `/story-done` review:

1. **Story AC-1 factual correction**: Original AC-1 specified `config/features=PackedStringArray("4.6", "Forward Plus Mobile")`. Verified against Godot 4.6 demo projects (`godot-demo-projects/3d/platformer` and `misc/joypads`) — 4.6 does NOT store renderer tags in `config/features`; renderer is selected exclusively via `rendering/renderer/rendering_method`. AC corrected to `PackedStringArray("4.6")`.

2. **GdUnit4 install method changed submodule → vendored**: Initial approach was git submodule per tests/README.md original guidance. Submodule approach broken because gdUnit4's GitHub repo contains the addon at a nested `addons/gdUnit4/` path — incompatible with Godot's expectation of flat `addons/<name>/plugin.cfg`. Pivoted to vendored install: downloaded release tarball v6.1.2, extracted only the inner `addons/gdUnit4/` subtree to project's `addons/gdUnit4/`. Version pinned via `addons/gdUnit4/VERSION.txt`.

3. **Legacy `tests/gdunit4_runner.gd` wrapper deleted**: The pre-existing wrapper (authored 2026-04-18, never runnable because no Godot was installed) had two unfixable issues for gdUnit4 v6.x: (a) referenced the old `GdUnit4CliRunner.gd` name which was renamed to `GdUnitCmdTool.gd` in v6.0; (b) our wrapper and CmdTool both extend `SceneTree`, so one cannot cleanly delegate to the other. Replaced with direct CmdTool invocation documented in `tests/README.md` and story-000 AC-2. Also fixed a pre-existing GDScript parse error in the legacy wrapper (Python-style implicit string concatenation) before deletion — kept as a historical observation.

4. **`--ignoreHeadlessMode` flag required**: gdUnit4 v6.x refuses headless runs by default (warns that `InputEvent`-based tests don't work). Our test suite is pure logic/integration (no UI simulation), so ignoring the warning is safe. The flag is documented in `tests/README.md` and the story's AC-2.
